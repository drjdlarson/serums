serums.distances.calculate_ospa2

calculate_ospa2(est_mat, true_mat, c, p, win_len, core_method=SingleObjectDistance.MANHATTAN, true_cov_mat=None, est_cov_mat=None)[source]

Calculates the OSPA(2) distance between the truth at all timesteps.

Notes

This calculates the OSPA-on-OSPA, or OSPA(2) metric as defined by [1] and further explained in [2]. It can be thought of as the time averaged per track error between the true and estimated tracks. The inner OSPA calculation can use any suitable OSPA distance metric from calculate_ospa()

Parameters:

  • est_mat (S x T x N numpy array) – Numpy array of state dimension by number of timesteps by number of objects for times/objects which do not exist use a value of np.nan for all state dimensions (i.e. if object 1 at timestep 2 does not exist then est_mat[:, 2, 1] = np.nan * np.ones(state_dim)). This corresonds to estimated states.

  • true_mat (S x T x N numpy array) – Numpy array of state dimension by number of timesteps by number of objects for times/objects which do not exist use a value of np.nan for all state dimensions (i.e. if object 1 at timestep 2 does not exist then true_mat[:, 2, 1] = np.nan * np.ones(state_dim)). This corresonds to the true states.

  • c (float) – Distance cutoff for considering a point properly assigned. This influences how cardinality errors are penalized. For \(p = 1\) it is the penalty given false point estimate.

  • p (int) – The power of the distance term. Higher values penalize outliers more.

  • win_len (int) – Number of timesteps to average the OSPA over.

  • core_method (enums.SingleObjectDistance, Optional) – The main distance measure to use for the localization component of the inner OSPA calculation. The default value of None implies enums.SingleObjectDistance.EUCLIDEAN.

  • true_cov_mat (S x S x T x N numpy array, Optional) – Numpy array of state dimension by state dimension by number of timesteps by number of objects for times/objects which do not exist use a value of np.nan for all state dimensions (i.e. if object 1 at timestep 2 does not exist then true_cov_mat[:, :, 2, 1] = np.nan * np.ones((state_dim, state_dim))). This corresonds to the true states, the object order must be consistent with the truth matrix, and is only needed for core methods enums.SingleObjectDistance.HELLINGER. The default value is None.

  • est_cov_mat (S x S x T x N numpy array, Optional) – Numpy array of state dimension by state dimension by number of timesteps by number of objects for times/objects which do not exist use a value of np.nan for all state dimensions (i.e. if object 1 at timestep 2 does not exist then est_cov_mat[:, :, 2, 1] = np.nan * np.ones((state_dim, state_dim))). This corresonds to the estimated states, the object order must be consistent with the estimated matrix, and is only needed for core methods enums.SingleObjectDistance.MAHALANOBIS. The default value is None.

Returns:

  • ospa2 (numpy array) – OSPA values at each timestep.

  • localization (numpy array) – Localization component of the OSPA value at each timestep.

  • cardinality (numpy array) – Cardinality component of the OSPA value at each timestep.

  • core_method (enums.SingleObjectDistance) – Method to use as the core distance statistic.

  • c (float) – Maximum distance value used.

  • p (int) – Power of the distance term used.

  • win_len (int) – Window length used.
